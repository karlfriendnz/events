## Product Details

### FriendlyManager Events Platform

- **Mapping**: single
- **Packages**: `.`
- **Description**: A unified sports organisation platform that handles the full lifecycle of events — authoring, bookings, registration, payments, physical access control, and cross-module reporting. Built for both National Sports Organisations (NSOs) and affiliated clubs, with NSO→Club event sharing, phased registration, a conditional pricing engine, and Xero integration.
- **Include paths**: whole package
- **Exclude paths**: none
- **Notes**: This is a greenfield project — no code exists yet. The full product specification lives at `FriendlyManager_Events_Upgrade_Spec.md`. The platform has six tightly-coupled modules (Events §4, Bookings §5, Finances §6, Public Registration Forms §7, Lighting & Access Control §8, Reporting §9) that share a common core: Event/Session, Bookable, Person, and Fee Rule engine entities. The user confirmed these should be treated as one product rather than six separate products. The worked example throughout the spec is a 5-Day Football Camp scenario, which exercises the majority of the system. Figma prototype available at the URL referenced in the spec.
